# 我们的浪漫收藏本 - 需求清单与优先级

## P0（必须优先做）
1. 云端账号体系与数据同步
2. 线上数据库落库与基础数据模型设计
3. 任务/抽卡/收藏核心数据 API
4. 双人关系绑定（情侣关系建立）
5. 数据迁移策略（本地缓存迁移到云端）
6. 基础异常处理与数据兜底

## P1（核心体验增强）
1. 自定义任务
2. 双人视角与角色分工
3. 抽卡概率与奖池配置后台化
4. 消息提醒 / 每日签到提醒
5. 收藏册详情页与卡片故事感增强
6. 分享成就/分享卡片

## P2（留存与传播）
1. 图片上传记录美好瞬间
2. 成就系统扩展
3. 排行榜/情侣 PK
4. 节日限定卡池/活动玩法
5. 周报/月报回顾

## P3（可选探索）
1. AI 生成浪漫任务/情书文案
2. 更复杂的成长体系
3. 多端数据互通（H5/小程序）
4. 运营后台

## 优先级原则
- 先保证“数据安全可持续”
- 再增强“情侣互动体验”
- 最后再考虑“传播与运营玩法”
