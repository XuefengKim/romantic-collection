# UIUX设计说明：我们的浪漫收藏本（微信小程序）

---

## 一、整体设计规范
延续原有H5浪漫温馨风格，适配微信小程序设计规范：

### 1. 色彩系统
| 类型 | 色值 | 使用场景 |
|------|------|----------|
| 背景渐变 | `linear-gradient(135deg, #ffeef2 0%, #ffc0cb 100%)` | 页面通栏背景 |
| 主色 | `#d6336c` | 标题、主按钮、导航栏 |
| 强调色 | `#f72585` | 统计数字、抽卡按钮、高亮 |
| 主文字 | `#4a3737` | 正文标题 |
| 辅助文字 | `#7a6464` | 次要说明文字 |
| 卡片底色 | `rgba(255, 255, 255, 0.9)` | 模块卡片底色 |
| **稀有度配色** | | |
| R | `#4299e1` | R等级卡片标签/边框 |
| SR | `#9f7aea` | SR等级卡片标签/边框 |
| SSR | `#f687b3` | SSR等级卡片标签/边框 |
| UR | `#d53f8c` + `box-shadow: 0 0 15px 3px rgba(213, 63, 140, 0.6)` | UR等级卡片（带发光效果） |

### 2. 字体与组件规范
- **字体层级**：导航标题18px(600)、页面标题22px(600)、模块标题16px(500)、正文14px(400)、辅助文字12px(400)、统计数字24px(700)
- **圆角统一**：模块/卡片16px，按钮12px
- **阴影**： `0 4px 12px rgba(214, 51, 108, 0.15)` 柔和阴影
- **页面间距**：页面边距16px，模块间距20px，内容内间距16px

## 二、页面设计与交互说明

### 📄 页面1：首页（数据看板）
**布局结构**：
1. 顶部留出状态栏安全区（适配iOS/Android）
2. 标题区居中：
   - 主标题：我们的浪漫收藏本（22px主色）
   - 副标题：收集点滴心意，兑换专属惊喜（14px辅助色，间距8px）
   - 小彩蛋：每一点心意，都是爱你的证明 💌（12px辅助色）
3. 数据卡片（通栏白色半透明）：双栏布局
   - 左：「累计心意」+ 高亮数字（强调色24px加粗）
   - 右：「待领惊喜」+ X次（强调色24px加粗）
4. 功能入口（垂直排列，间距16px）：
   - 「去打卡」主按钮（主色背景，白色文字）
   - 「浪漫收集册」次按钮（主色描边，透明背景）
   - *预留「排行榜」入口位置，当前MVP版本隐藏*
   - *有可抽次数时额外展示「开启惊喜盲盒 ✨」强调色按钮，带2s周期呼吸动画*

**交互逻辑**：点击按钮直接跳转对应页面，数据实时更新

---

### 📄 页面2：任务打卡页
**布局结构**：
1. 导航栏（返回+标题「任务打卡」）
2. 分类标签栏：「浪漫任务」/「日常点滴🧹」，差异化设计：
   - 「浪漫任务」：紫色背景（对应SR稀有度氛围）白字
   - 「日常点滴」：主色粉色背景白字 + 小扫把emoji
3. 任务列表（每个任务独立卡片，间距12px）：
   - 任务名称（16px深咖啡加粗）
   - 说明文字：已累计X点心意（12px辅助色）
   - *家务任务额外增加小字提示：完成打卡，积攒抽卡机会*
   - 右下角「打卡」按钮，固定120px宽度

**交互逻辑**：
- 切换标签展示对应分类任务
- 点击打卡：成功+1心意，按钮自动进入60s倒计时（显示剩余秒数，灰色不可点击）
- 累计满5点心意，自动增加1次抽卡机会，数据实时存在本地缓存
- 倒计时结束自动恢复可点击状态，防止重复打卡

---

### 📄 页面3：惊喜盲盒抽卡页
**布局结构**：
1. 导航栏（返回+标题「浪漫惊喜盲盒」）
2. 中央盲盒展示区：粉色礼盒图形+提示文字「你还有 X 次抽卡机会」，数字高亮强调色
3. 底部抽卡按钮：
   - 有次数：「开启惊喜盲盒 ✨」强调色背景，呼吸动画
   - 无次数：灰色禁用，提示「累计5点心意可抽卡」

**抽卡弹窗（核心交互）**：
- 触发方式：点击抽卡后弹出，带从小到大旋转弹簧动画（0.5s）
- 结构：半透明遮罩（点击关闭）+ 居中弹窗（80%宽度，圆角20px）
  - 顶部：「恭喜获得！」居中主色文字
  - 主体：展示抽到的卡片（带对应稀有度颜色+发光效果），包含稀有度标签、卡片名称、描述
  - 底部：「收下卡片」按钮，点击关闭弹窗
- 反馈：抽到SSR/UR增加短震动反馈，屏幕边缘增加对应稀有度颜色渐变闪烁效果，强化惊喜感
- 氛围：抽卡后飘出2-3个小爱心缓缓上升消失，增加浪漫氛围

---

### 📄 页面4：浪漫收集册
**布局结构**：
1. 导航栏（返回+标题「浪漫收集册」）
2. 两列网格布局（间距12px，边距16px），共6个卡片位，对应6张固定奖券
- **未收集卡片**：浅白色半透明，居中显示大浅粉色问号，底部文字「未收集」
- **已收集卡片**：白色半透明，顶部对应稀有度标签，然后卡片名称+描述，右下角标注「已收集：X张」

**交互逻辑**：
- 点击已收集卡片可弹出详情弹窗（和抽卡结果弹窗一致），数据实时同步
- **满级成就彩蛋**：当6张卡片全部集齐后，自动弹出「浪漫满级成就」弹窗：
  - 标题：💑 恭喜你们！浪漫收集满级啦 💑
  - 正文：感谢你们把每一件小事都变成浪漫，继续加油，永远爱下去~
  - 按钮：「分享成就」（预留后续云端分享能力）+「再攒一波」

## 三、补充设计调整（结合产品背景）

### 一、设计思路强化
基于创始人灵感：**用抽卡集卡的游戏化，把做家务变成情侣间有趣的收集游戏，降低矛盾，增加情趣**。我们在设计上强化「游戏感」和「情侣情趣氛围」。

### 二、细节设计优化
1. **任务分类视觉强化**：如上文任务打卡页设计，强化家务分类认知
2. **收集完成彩蛋设计**：全收集弹出满级成就弹窗
3. **氛围细节加分**：抽卡飘爱心、SSR/UR闪烁震动、数据看板小彩蛋
4. **未来扩展设计预留**：首页排行榜入口、数据存储预留云端字段、收集册支持扩展更多卡片

### 三、UX体验贴合场景
- 所有操作流程都是3步以内完成，不复杂：打卡 → 攒心意 → 抽卡，降低学习成本
- 家务任务固定6个，MVP版本足够轻量化，不会给用户造成负担，符合「日常随手做」的定位
- 情侣双人使用，所有界面文案都是偏向「我们」的口吻，强化二人专属感，而不是冰冷的工具感
- 离线可用：所有数据存储在微信小程序本地缓存，无需登录/服务器，满足双人本地使用需求
- 安全适配：预留iOS状态栏、微信小程序底部导航安全区，适配全尺寸手机

---

整体设计完全满足MVP版本需求，同时为后续功能扩展预留了设计空间，贴合创始人「用游戏化把家务变浪漫」的初衷。
