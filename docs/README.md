### 需求
- `docs/product/requirements/PRD.md`：产品需求文档

## 3. 文档整理原则
1. 产品相关：放 `docs/product/`
2. 研发相关：放 `docs/development/`
3. 规划类文档：放在各自分类下的 `planning/` 或 `plans/`
4. 后续如增加测试、部署、运维文档，可继续扩展为：
   - `docs/development/testing/`
   - `docs/development/deployment/`
   - `docs/development/operations/`
